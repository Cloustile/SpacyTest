## 更新日志

### 2026-03-19

#### ✨ 新增功能
- **知识图谱上下文增强**
  - 在 `preKG.txt` 中为每个节点（实体）添加上下文句子
  - 在 `preKG.txt` 中为每条边（关系）添加上下文句子
  - 上下文信息同时保存在 JSON 格式中，便于程序化处理
  
- **语言支持优化**
  - 支持中文和英文处理（可通过配置切换）
  - 添加语言切换脚本 `switch_language.py`

#### 📝 文件结构更新
```
输入:
  - data.txt (数据文本)
  - question.txt (问题文本)

输出:
  - dataKB.txt (数据知识库 - 包含所有实体和关系)
  - questionKB.txt (问题知识库 - 包含问题中的实体和关系)
  - preKG.txt (知识图谱 - 包含上下文信息的筛选后知识)
  - answer.txt (LLM 生成的答案)
```

#### 🔧 代码改进
- `main.py`: 
  - `build_knowledge_graph()` 函数增强，添加 context 字段
  - 节点现在包含出现该实体的所有句子
  - 边包含表达该关系的具体句子

## preKG.txt 文件格式示例

### 之前的格式（无上下文）
```
=== Nodes (Entities) ===
[n0] 北京 (Type: GPE)
[n1] 中国 (Type: GPE)

=== Edges (Relations) ===
n0 --[是]--> n1
  Label: 北京 -> 是 -> 首都
```

### 现在的格式（带上下文）✨
```
=== Nodes (Entities) ===
[n0] 北京 (Type: GPE)
  Context:
    - 北京是中国的首都。
    - 作者曾多次访问北京。
    
[n1] 中国 (Type: GPE)
  Context:
    - 北京是中国的首都。
    - 中国有着悠久的历史。

=== Edges (Relations) ===
n0 --[是]--> n1
  Label: 北京 -> 是 -> 首都
  Context: 北京是中国的首都。
```

### JSON 格式（程序可用）
```json
{
  "nodes": [
    {
      "id": "n0",
      "label": "北京",
      "type": "GPE",
      "context": [
        "北京是中国的首都。",
        "作者曾多次访问北京。"
      ]
    }
  ],
  "edges": [
    {
      "source": "n0",
      "target": "n1",
      "relation": "是",
      "label": "北京 -> 是 -> 首都",
      "context": "北京是中国的首都。"
    }
  ]
}
```

## 使用上下文信息的优势

1. **更好的可解释性** 📖
   - 可以看到每个实体在原文中出现的具体语境
   - 理解关系的来源和背景

2. **LLM 回答更准确** 🎯
   - 知识图谱中的上下文为 LLM 提供了更多细节
   - 减少幻觉，提高答案的准确性

3. **便于调试和验证** 🔍
   - 可以追溯每个知识点的来源
   - 快速定位提取错误

4. **支持后续扩展** 🚀
   - JSON 格式便于其他程序读取和处理
   - 可以轻松导入图数据库（如 Neo4j）
